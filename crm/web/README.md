# عقّار CRM — الواجهة (Next.js + Tailwind + shadcn)

الواجهة البصرية المتكاملة لنظام الـ CRM العقاري، ببيانات تجريبية (لا تحتاج خادماً)
لضبط تجربة المستخدم أولاً. مبنية بفلسفة **البساطة الممتنعة بنمط Notion**:
مساحات بيضاء واسعة، خطوط نظيفة، صفر زحمة بصرية، وسحب وإفلات في لوحة المبيعات.

> مستقلّ تماماً عن أي مشروع آخر في المستودع. قاعدة البيانات وسياسات العزل في
> [`../supabase`](../supabase)، وهذه الواجهة تعكس نفس النموذج ببيانات وهمية.

## التشغيل

```bash
cd crm/web
npm install
npm run dev        # http://localhost:3000
```

للبناء والتشغيل الإنتاجي: `npm run build && npm run start`.

## الواجهتان (Dual Interface)

يتبدّل المستخدم بينهما من مفتاح البوابة أعلى القائمة الجانبية.

### بوابة التسويق (فريق الوكالة)
| المسار | الشاشة |
|---|---|
| `/marketing` | لوحة تحكم شاملة — مؤشرات كل الشركات، مصادر الليدز |
| `/marketing/companies` | الشركات — كل شركة بوابة معزولة بهويتها |
| `/marketing/assistant` | **المساعد الذكي** — رفع Excel بأوامر طبيعية + بطاقة معاينة قبل الحفظ |
| `/marketing/rotation` | **توزيع الليدز** — المدة والعدد وتوصيات الذكاء الاصطناعي مع الأسباب |

### واجهة المبيعات (المستخدم النهائي)
| المسار | الشاشة |
|---|---|
| `/sales` | **خط الأنابيب** — لوحة Kanban بالسحب والإفلات + درج تفاصيل الليد |
| `/sales/supermarket` | **سوق العقارات** — مخزون المطوّرين بفلترة فورية وحجز مباشر |

## لغة التصميم

- ألوان محايدة دافئة بنمط Notion، معرّفة كمتغيرات CSS في `app/globals.css`.
- خطوط عربية نظيفة (IBM Plex Sans Arabic كتحسين تدريجي) + RTL أصيل من الجذر.
- مكوّنات shadcn/ui مملوكة داخل `components/ui` — يسهل تعديلها.
- سحب وإفلات عبر `@dnd-kit`، أيقونات خطية رفيعة عبر `lucide-react`.

## البنية

```
crm/web/
├── app/
│   ├── layout.tsx              الجذر — RTL + الخط
│   ├── marketing/…             بوابة التسويق (4 شاشات)
│   └── sales/…                 واجهة المبيعات (شاشتان)
├── components/
│   ├── app-shell.tsx           القائمة الجانبية + مبدّل البوابة + الشريط العلوي
│   ├── ui/                     مكوّنات shadcn (button, card, dialog…)
│   ├── kanban/                 لوحة السحب والإفلات + بطاقة الليد
│   ├── assistant/              محادثة المساعد + بطاقة معاينة الاستيراد
│   ├── supermarket/            سوق العقارات
│   ├── rotation-panel.tsx      لوحة تحكم التوزيع
│   └── lead-drawer.tsx         درج تفاصيل الليد
└── lib/
    ├── mock-data.ts            بيانات تجريبية تعكس سكيما المرحلة 1
    └── utils.ts                cn + تنسيق العملة
```

## الخطوة التالية

ربط هذه الواجهة بقاعدة بيانات Supabase (في `../supabase`) عبر Server Components
واستدعاءات مؤمّنة بسياسات RLS — استبدال `lib/mock-data.ts` باستعلامات حقيقية.
